Analys

Mina förväntningar innan kursen har varit att lära mig åtminstone grundläggande webprogrammering samt hur dagens systemutvecklingsprojekt genomförs med hjälp av Agila utvecklingsmetoder som SCRUM. Detta ämne har varit väldigt nytt för mig och hade endast lekt runt lite i HTML innan medans CSS och JavaScript var helt outforskade för mig. Agila utvecklingsmetoder var också ett helt nytt begrepp för mig. Kursen började med grundläggande lektioner i Webprogrammering där vi lärde oss HTML och CSS. För att förankra dessa kunskaper så gjorde vi uppgifter på w3schools.com som erbjuder guider, quiz och uppgifter gratis inom de vanligaste programmeringsspråken men även andra områden som till exempel Git och tillgänglighet för hemsidor. Till dessa uppgifter på w3schools fanns två inlämningsuppgifter där man bedömdes efter hur mycket man hade klarat till varje delmoment. För mig var uppgifterna och guiderna mest givande på w3schools men det som fick de flesta poletterna att trilla ner för mig var att jobba på en egen hemsida och försöka själv så gott det gick. När jag fastnade på något så gick det i princip alltid att hitta en lösning till problemet genom att använda valfri sökmotor. För mer avancerade och större problem som att göra hemsidan responsiv så letade jag upp en guide på Youtube.

Kursens huvudsakliga examinationsuppgift är att skapa en egen hemsida som handlar om alla de ämnen som kursen berör är även denna text är publicerad på. Som utgångspunkt hade vi en pdf som innehöll alla kraven. För att organisera projektet så introducerades vi till Azure DevOps som är en online tjänst som erbjuds av Microsoft där man kan planera sina projekt. Det finns alla verktyg där som behövs om man jobbar med SCRUM då man kan skapa backloggar och sprints som sedan går att hantera ytterligare i Boards. Jag började lite sent med min hemsida och missade att hantera min första sprint som vi lade upp tillsammans under lektionstid, den andra sprinten använde jag mig av verktygen i Azure DevOps där jag höll koll på hur mycket det fanns kvar att göra i projektet via Boards fliken. I Azure DevOps finns också möjlighet till versionshantering av projekt. Jag använde dock Github för mitt projekt då jag var lite mer bekväm med att använda Github. Git och Github gjorde det väldigt smidigt att backa till äldre versioner av mitt projekt i de fall jag inte blev nöjd med ändringarna som jag gjorde och var även väldigt användbart då jag programmerar på två olika datorer och det endast behövs några få kommandon för att göra transitionen mellan datorerna smärtfritt.

För min del flöt projektet på rätt bra då samspelet mellan HTML, CSS och JavaScript klickade ganska snabbt för mig. Under kursens gång lärde vi oss lite om Bootstrap som är ett ramverk för webprogrammering där det finns en mängd färdiga mallar för till exempel en meny eller footer. I mitt projekt har jag inte använt mig av Bootstrap utan jag skapade det mesta från scratch. Anledningen till det var att jag ville säkerställa att jag förståd all kod som jag skrev till min hemsida även om det innebar extra arbete för mig. Min plan var att skapa en rätt minimalistisk hemsida vilket jag känner jag har lyckades med. Kursen gav mig en blodad tand för webbprogrammering och jag ser fram emot att fördjupa mina webbutvecklingskunskaper i den kommande JavaScript kursen.
